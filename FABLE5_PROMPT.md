# 이벤트플러스 MVP 구현 요청

## 목표
행사 공간 사진을 업로드하면 GPT가 사진 속 물품(사람, 의자, 테이블, 배너 등 임시 설치물)을 탐지하고, 사용자가 결과를 검토·수정한 뒤, 해당 물품이 제거된 빈 공간 이미지를 생성해서 보여주는 웹앱을 만들어줘.

풀 서비스(대시보드, 로그인, 프로젝트 저장, 공유 등)는 이번에 만들지 않는다. **"사진 넣고 → 물품 찾고 → 사용자가 확인/수정하고 → 지운 결과 보여주기"** 이 한 사이클만 잘 도는 1페이지짜리 프로토타입이면 된다.

## 기술 스택
- Next.js (App Router) + TypeScript + Tailwind CSS
- 이미지 처리/스타일링은 자유롭게 선택
- OpenAI API만 사용 (`OPENAI_API_KEY` 환경변수, 이미 `.env.local`에 설정되어 있음)
- DB 없음. 세션 상태(React state)로만 관리, 새로고침하면 초기화되어도 무방

## 개발 및 배포 환경

### 저장소
- 이미 GitHub 저장소가 만들어져 있고 원격 연결되어 있음: `https://github.com/jsulim/event-plus.git`
- 로컬 경로: `C:\intween\eventsolution` (이 폴더 안에 바로 Next.js 프로젝트를 생성할 것 — 하위 폴더로 새로 만들지 말 것)
- `.git`, `.gitignore`, `README.md`, `FABLE5_PROMPT.md`, `.env.local`은 이미 존재하니 그대로 유지하고 그 위에 Next.js 스캐폴딩을 추가할 것
- `.env.local`은 `.gitignore`에 이미 등록되어 있어 커밋되지 않음 — 이 안의 `OPENAI_API_KEY` 값은 코드에 하드코딩하지 말고 반드시 `process.env.OPENAI_API_KEY`로만 참조할 것
- 작업 완료 후 변경사항은 `main` 브랜치에 커밋 후 `git push`로 반영 (강제 푸시 금지)

### 로컬 개발
- `npm install` → `npm run dev` → `http://localhost:3000` 접속
- Node.js 버전은 20 이상 가정

### 배포 (Vercel)
- Vercel에 위 GitHub 저장소(`jsulim/event-plus`)를 import해서 배포할 예정 (아직 연결 전이면 사용자가 직접 Vercel 대시보드에서 연동)
- `main` 브랜치 push 시 프로덕션 자동 배포, 다른 브랜치/PR은 프리뷰 URL 자동 생성 (Vercel 기본 동작이므로 별도 설정 불필요)
- 환경변수(`OPENAI_API_KEY`)는 로컬 `.env.local`과 별개로 Vercel 프로젝트 설정(Environment Variables)에도 동일하게 등록해야 배포본에서 동작함 — 이 등록은 사용자가 Vercel 대시보드에서 직접 수행
- **주의**: OpenAI 이미지 생성(`/api/generate`) 호출은 몇 초~십수 초 걸릴 수 있음. Vercel 서버리스 함수의 기본 실행시간 제한에 걸리지 않도록, 해당 API 라우트는 `export const maxDuration`을 넉넉히 설정(예: 60초)하거나 요청 자체의 timeout을 여유있게 잡을 것
- 서버 전용 코드(OpenAI 호출)는 반드시 API Route(서버 사이드)에서만 실행하고, 클라이언트 컴포넌트에 API 키가 노출되지 않도록 할 것

## 화면 흐름 (1페이지)
1. **업로드**: 드래그앤드롭 또는 파일선택으로 행사 공간 사진 업로드, 미리보기 표시
2. **분석하기 버튼** 클릭 → 로딩 상태 표시 → 분석 API 호출
3. **분석 결과 표시**: 원본 이미지 위에 탐지된 객체들을 bbox(사각형)로 오버레이 표시. 색상 구분:
   - 제거 대상(remove): 빨강/주황 반투명
   - 보존 대상(preserve): 파랑 반투명
   - 확인 필요(uncertain): 노랑 반투명
   - 옆에 객체 목록(라벨 + 분류 상태) 표시, 목록 항목 클릭 시 해당 박스 강조
4. **사용자 수정**: 객체 목록에서 각 항목의 분류를 클릭으로 토글 변경 가능 (제거 ↔ 보존 ↔ 확인필요). 확인필요 상태인 객체는 최종적으로 제거 대상에 포함되지 않는다.
5. **빈 공간 생성 버튼** 클릭 → 로딩 상태 표시 → 생성 API 호출 (현재 "제거"로 표시된 객체들만 대상)
6. **결과 비교**: Before/After를 좌우 나란히 또는 슬라이더로 비교 표시. 결과 이미지 다운로드 버튼 제공

## API 설계

### `POST /api/analyze`
- 입력: 업로드된 이미지 (base64 또는 multipart)
- 처리: OpenAI `gpt-4o` (vision) 모델에 이미지를 보내고, 아래 분류 기준에 따라 물품을 탐지·분류하도록 요청. 응답은 JSON 구조화 출력(structured output / response_format json_schema)으로 강제해서 파싱 안정성 확보
- 응답 예시:
```json
{
  "objects": [
    { "id": "obj_1", "label": "의자", "classification": "remove", "bbox": [x, y, width, height], "confidence": 0.9 }
  ]
}
```
- bbox 좌표는 원본 이미지 픽셀 기준 (또는 0~1 정규화 비율로 통일 — 프론트 오버레이 렌더링과 맞춰서 구현)

### `POST /api/generate`
- 입력: 원본 이미지 + 최종 확정된 제거 대상 객체들의 bbox 목록
- 처리:
  1. 제거 대상 bbox들을 합쳐서 마스크 이미지 생성 (검은 배경에 흰색 사각형들, PNG, 알파 채널로 마스크 표현)
  2. OpenAI 이미지 편집 API(`gpt-image-1`, images edit endpoint)에 원본 이미지 + 마스크 + 프롬프트("이 마스크 영역의 물체를 제거하고 배경(벽/바닥/천장)을 자연스럽게 복원해줘, 건축 구조는 유지해줘")를 전달
  3. 결과 이미지를 base64로 응답
- 응답 예시:
```json
{ "resultImage": "data:image/png;base64,..." }
```

## 객체 분류 기준 (analyze 프롬프트에 그대로 포함)

**제거 대상(remove)**: 사람, 의자, 테이블, 소파, 배너, 현수막, 전시대, 행사 부스, 포토월, 이동식 안내데스크, 이동식 무대, 음향장비, 스탠드형 조명, 모니터, 이동식 스크린, 박스, 케이블, 카트, 쓰레기통, 장식물, 임시 가설물

**보존 대상(preserve)**: 벽, 바닥, 천장, 기둥, 창문, 출입구, 계단, 엘리베이터, 매립조명, 고정형 천장조명, 고정형 무대, 빌트인 가구, 소방시설, 콘센트, 고정형 안내데스크, 건축 구조 일반

**확인 필요(uncertain)**: 파티션, 트러스, 무대, 대형 모니터, 스크린, 안내데스크, 대형 화분, 장기 설치 전시물, 장식 구조물, 반고정형 벽체, 고정 여부 판단 어려운 설치물

목록에 없는 새로운 물체를 만나면: 이동 가능해 보이면 remove, 건축 구조물이면 preserve, 애매하면 uncertain으로 분류하도록 프롬프트에 명시.

## 사용자 수정 이력 저장 (가능하면 구현, 필수는 아님)
나중에 자체 모델 파인튜닝에 쓸 수 있도록, "GPT의 원래 분류 vs 사용자가 최종 수정한 분류"의 차이를 세션 내에서라도 콘솔 로그나 로컬 JSON 파일로 남겨두면 좋음. 이번 MVP에서 DB 연동은 하지 않는다.

## 이번에 하지 않는 것 (스코프 아웃)
- 로그인/회원가입
- 프로젝트 대시보드, 여러 프로젝트 관리
- DB 저장, 공유 링크, 댓글, 피드백 평점
- 브러시로 마스크 직접 그리기 (bbox 토글 수정만 지원)
- 2D 도면, AI 배치 추천, 견적/발주

## 완료 기준
- 임의의 행사 공간 사진을 업로드하면 실제로 GPT가 물품을 탐지해서 화면에 보여줌 (하드코딩된 고정 결과 아님)
- 사용자가 분류를 수정할 수 있음
- "빈 공간 생성" 클릭 시 실제로 OpenAI 이미지 편집 API를 호출해서 물품이 제거된 결과 이미지가 나옴
- Before/After 비교 화면에서 결과 확인 가능
- README에 실행 방법(`npm install`, `npm run dev`, `.env.local` 설정법) 명시
